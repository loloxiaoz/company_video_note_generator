# 大语言模型之外，AI Agent 究竟需要什么才能真正工作？

你有没有想过，为什么市面上那么多 AI 聊天机器人，却很少有真正能帮你完成复杂任务的？这不是巧合。当 Salesforce 首席 AI 科学家 Silvio Savarese 在访谈中揭开 AgentForce 的设计理念时，他道出了一个关键真相：**构建能够实际工作的 AI Agent，远不止是调用一个大语言模型那么简单**。

## Connection：从聊天到行动的鸿沟

当你打开 ChatGPT 构建一个自定义 GPT 时，你可能会觉得这就是 AI Agent 的全部了。输入指令，设定角色，添加一些知识库文件，然后就能对话了。但 Savarese 提出了一个更深层的问题：**如果 AI 只能聊天，它如何真正融入你的业务流程？**

作为 Salesforce 的首席 AI 科学家，Savarese 的角色并非仅仅发表论文或开发算法。他的使命是"观察 AI 如何在企业中转型，如何让 AI 成为业务变革和新产品创造的核心"。这意味着他必须思考的不是 AI 能做什么，而是**企业需要 AI 做什么**。

这个视角的转变至关重要。十年前，Salesforce 设立首席科学家这个职位时，就明确了一个方向：研究必须服务于产品，技术必须解决实际问题。从四年前开始构建大语言模型（那时 ChatGPT 还未问世），到开发 Einstein Trust Layer 确保 AI 输出的安全性，再到去年推出的 Atlas Reasoning Engine 规划引擎，每一步都在回答同一个问题：**如何让 AI 从实验室走向生产环境？**

## Conflict：聊天机器人的局限性

这里就出现了核心矛盾。大多数人对 AI Agent 的理解停留在"能对话的 AI"层面，但 Savarese 明确指出：**AgentForce 不仅仅是一个聊天机器人**。

想象一下，你让一个只会聊天的 AI 帮你处理客户投诉。它可能会给你很好的建议，甚至生成一封得体的回复邮件。但它能自动查询客户的历史订单吗？能调用退款系统吗？能更新 CRM 记录吗？能在处理完后自动通知相关部门吗？答案是不能。

这就是问题所在。**真正的业务场景需要的不是对话能力，而是执行能力**。一个客服 Agent 需要访问客户数据库，一个销售 Agent 需要整合 CRM 系统，一个物流 Agent 需要对接库存管理。单纯的语言模型无法完成这些任务，因为它缺少关键的系统组件。

Savarese 用一个类比来解释 AgentForce 的设计理念：**它是一个由多个专业 Agent 组成的平台**，每个 Agent 专注于不同的任务。这不是一个万能的聊天机器人，而是一个可以根据业务需求定制的 Agent 生态系统。

## Change：构建真正的 AI Agent 系统

那么，一个真正能工作的 AI Agent 需要什么？Savarese 给出了清晰的答案：**它需要的不仅是大脑，还需要记忆、工具和推理能力**。

首先是**记忆系统**。AgentForce 能够检索信息、访问数据、查阅文件并参与持续的对话。这不是简单的上下文窗口，而是一个完整的知识管理系统。通过 RAG（检索增强生成）等技术，Agent 可以在需要时调用相关信息，而不是把所有知识都塞进模型里。想想看，一个客服 Agent 需要记住成千上万的产品细节和政策条款，如果没有外部记忆系统，这根本无法实现。

其次是**工具调用能力**。Savarese 特别强调了 function calling（函数调用）的重要性，这让 Agent 能够与外部系统交互。这意味着 Agent 不再局限于生成文本，它可以执行实际操作：发送邮件、更新数据库、触发工作流、调用 API。**这是从"知道怎么做"到"真正去做"的关键一步**。

最后是**推理引擎**。这就是 Savarese 提到的 Atlas Reasoning Engine。它让 Agent 能够将复杂任务分解为多个步骤，进行规划和决策。比如处理一个退款请求，Agent 需要：验证订单、检查退款政策、计算金额、发起退款流程、通知客户、更新记录。这个过程需要逻辑推理和任务规划，而不仅仅是文本生成。

这三个组件共同构成了一个完整的 Agent 系统。**大语言模型只是其中的一部分——它是 Agent 的"大脑"，但记忆、工具和推理能力才让这个大脑真正有用**。

## Catch：重新定义 AI Agent

回到最初的问题：你应该构建一个 GPT 还是一个 AgentForce？现在答案应该很清楚了。

如果你只需要一个对话助手，提供建议和信息，那么自定义 GPT 就足够了。但如果你需要 AI 真正参与业务流程，自动化复杂任务，与现有系统集成，**你需要的是一个完整的 Agent 系统**。

Savarese 的洞察为我们揭示了 AI Agent 的未来方向。这不是关于谁的模型更大、谁的对话更流畅，而是关于**如何构建一个能够在真实世界中可靠工作的系统**。记忆让 Agent 能够访问知识，工具让 Agent 能够执行操作，推理让 Agent 能够处理复杂任务。

这也解释了为什么 Salesforce 花了十年时间构建这个平台。他们不是在追逐最新的模型基准，而是在解决一个更根本的问题：**如何让 AI 成为企业的生产力工具，而不仅仅是一个聊天玩具**？

当你下次考虑部署 AI Agent 时，不妨问问自己：我需要的是一个能聊天的 AI，还是一个能做事的 AI？如果是后者，那么你需要的不仅是一个模型，而是一个完整的系统——就像 AgentForce 所展示的那样。

# 为什么企业AI应用不能只靠大语言模型？

你可能已经注意到，ChatGPT和其他AI工具正在改变我们的工作方式。但如果你是一家企业，想要真正将AI融入业务流程，仅仅使用一个大语言模型就够了吗？答案可能会让你意外。

## Connection：不只是模型那么简单

当你考虑为企业部署AI时，可能会想：我能不能直接使用ChatGPT的Agent Builder，或者把Salesforce的数据接入ChatGPT？这个问题看似简单，但实际上揭示了企业AI应用的核心挑战。

大语言模型确实强大，但它只是完整解决方案的一部分。就像你不能仅凭一颗强劲的引擎就造出一辆汽车，企业AI也需要更完整的基础设施。Salesforce在其最大的开发者大会上展示的案例——无论是Williams Sonoma、Pepsi这样的大型企业，还是中小型开发商——都证明了这一点。

要构建真正有效的企业AI Agent，你需要四个核心功能协同工作：首先是推理能力，让Agent能够处理复杂任务并做出决策；其次是记忆系统，使Agent能够记住过去的互动和上下文；第三是Actuators（执行器），将任务转化为实际行动，比如发送邮件、创建网站或安排预约；最后是交互接口，让Agent通过语音、文字等多种方式与人交流。

## Conflict：数据孤岛与信任危机

这里就出现了关键矛盾：Agent Builder本身功能再强大，如果没有正确的数据和内容支撑，也只是空中楼阁。

想象一下，你的企业可能有分散在不同系统中的客户数据——销售部门有一套客户管理数据，商务部门又有另一套。如果这些触点没有连接起来，如果你没有正确的统一数据视图，AI就无法有效运作。这就是为什么仅仅使用一个通用的Agent Builder，而不配合完整的企业基础设施，很难真正解决问题。

更严峻的挑战是信任问题。你可能听说过AI的"幻觉"现象——大语言模型有时会生成看似合理但实际错误的信息。这些模型可以完成一些令人惊叹的复杂任务，比如通过理论考试成为金融顾问，但在一些看似简单的常规业务中却可能出错。

考虑这样一个场景：客户向银行咨询贷款利率，AI给出了错误的利率信息。这不仅仅是一个小失误，而可能导致严重的法律和财务后果，损害客户和金融机构之间的信任。在企业环境中，这种错误是不可接受的。

## Change：企业沙盒的解决之道

如何在保持创新的同时确保AI的可靠性？答案是企业沙盒环境。

这不是你想象中的那种简单测试环境或自动化游戏环境。企业沙盒是一个让AI在真实企业场景中进行压力测试的空间，使用企业自己的数据、流程和业务逻辑。

即使是GPT-5这样的下一代模型，我们仍然会看到一些不稳定的结果。这就是为什么需要从纯粹的模型训练转向实际的企业验证。最好的方法就是让这些AI系统在企业沙盒中接受考验，了解它们在真实世界中的表现和失败模式。

以客服语音Agent为例，在沙盒环境中，你可以测试它如何处理不同的口音（比如印度口音）、背景噪音、不完整的信息，以及各种边缘情况。这些都是可能导致AI失败的场景，而你需要让它们变得足够稳定。

测试过程是这样的：邀请真实的B2B和B2C组织参与，让真正的专业人员进行实际测试。这个挑战在于缩小测试环境与真实世界之间的差距。比如测试一个客服Agent时，你会设置各种场景——有人可能会提出刁钻的问题，有人可能会测试系统的边界——就像在游戏中与测试对手对战，找出可能的失败点。

## Catch：构建可信赖的企业AI

那么，你现在应该做什么？

如果你正在考虑为企业部署AI，记住这不是简单地选择一个大语言模型那么简单。你需要：

**建立完整的基础设施**。确保你的数据是统一的、可访问的，不同系统之间的触点是连接的。没有这个基础，再强大的AI模型也无法发挥作用。

**重视信任和质量**。在企业环境中，AI的准确性和可靠性比炫酷的功能更重要。一个稳定可靠、能够持续提供正确答案的AI，远比一个偶尔表现惊艳但经常出错的系统更有价值。

**利用企业沙盒进行验证**。在将AI推向实际应用之前，在受控环境中进行充分测试。让它面对真实的业务场景、真实的用户，找出并修复潜在问题。

AI技术的进展速度确实令人惊叹，但真正的突破将发生在企业级应用的成熟度上。当越来越多的企业能够安全、可靠地部署AI，将其深度融入业务流程时，我们才能看到AI真正改变商业世界的力量。

这不是一个关于选择哪个AI工具的问题，而是关于如何构建一个完整、可信赖的企业AI生态系统的问题。你准备好迎接这个挑战了吗？

# 个人AI代理会如何改变我们的商业交互方式？

你有没有想过，未来某一天，你不再需要亲自打电话给银行、保险公司或各种服务机构？取而代之的是，你的个人AI代理会代替你完成这一切。这听起来像科幻小说，但这个未来可能比你想象的更近。

## Connection：从贷款申请说起

让我们从一个简单的场景开始。假设你需要申请一笔贷款。在今天，你需要填写冗长的表格，打电话给银行，或者访问他们的网站，经历漫长而繁琐的申请流程。但在不久的将来，这一切都会改变。

你不会再亲自联系贷款机构。相反，你只需要告诉你的个人AI代理："我需要一笔贷款。"然后，你的AI代理会自动联系贷款机构的AI代理，两个AI系统会直接进行交流和协商。你所需要做的，只是在最后确认结果。

这不是遥不可及的幻想。随着AI技术的快速发展，我们正在进入一个由AI代理主导商业交互的新时代。

## Conflict：当机器开始代替我们说话

这种转变带来了一个根本性的挑战：当AI代理开始代表我们进行交流时，整个商业生态系统需要重新构建。

想象一下这个场景：你的AI代理需要与银行的AI代理对话，保险公司的AI代理需要与医院的AI代理交换信息，你的购物AI代理需要与各个商家的AI代理讨价还价。这意味着什么？这意味着我们需要建立一套全新的通信协议和标准。

就像互联网诞生之前，我们需要发明TCP/IP协议来让计算机之间能够通信一样，我们现在需要为AI代理之间的交互创建新的"交通规则"。这些AI代理需要一种标准化的方式来相互识别、验证身份、交换信息并达成协议。

更重要的是，我们面临着一个前所未有的问题：如何确保这些代表我们的AI真正理解我们的需求和偏好？如何保证它们在谈判中维护我们的最佳利益？如何建立信任机制，让我们放心地将重要决策委托给机器？

## Change：重塑商业交互的基础设施

这种变革将彻底改变我们所知的商业世界。我们需要重新思考整个商业基础设施的架构。

首先，我们需要建立AI代理之间的通信标准。这可能类似于互联网早期的协议发展，但更加复杂。这些协议不仅需要处理数据传输，还需要处理意图理解、偏好匹配、谈判策略等高层次的交互。

其次，身份验证和授权系统将变得至关重要。你的AI代理需要能够证明它确实代表你，而对方的AI代理也需要证明它代表的是合法的商业实体。这需要建立一套可靠的数字身份体系。

第三，我们需要新的监管框架。当AI代理代表我们签订合同、做出承诺时，法律责任如何界定？如果AI代理做出了不符合我们意愿的决定，谁来承担责任？这些都是需要解决的新问题。

想象一下，这就像从马车时代过渡到汽车时代。我们不仅需要发明汽车，还需要建设公路、制定交通规则、设立加油站、培训驾驶员。AI代理的普及将引发同样规模的基础设施革命。

## Catch：拥抱激动人心的未来

这场变革令人兴奋。我们正站在一个新时代的门槛上，一个AI代理将成为我们日常生活中不可或缺的助手的时代。

在这个未来中，你不再需要花费数小时比较不同的保险方案、寻找最优惠的机票价格或处理繁琐的行政事务。你的个人AI代理会在后台默默完成这一切，只在需要你做出关键决策时才会寻求你的意见。

这不仅仅是效率的提升，更是一种生活方式的转变。我们将有更多时间专注于真正重要的事情——创造、思考、与他人建立有意义的联系。那些重复性的、程序化的任务将被AI代理接管。

全世界都将因此而改变。商业模式会演化，新的服务和产品会涌现，我们与技术的关系也会变得更加紧密和自然。

这是一段激动人心的旅程，我们都是这场变革的见证者和参与者。虽然前方还有许多挑战需要克服，但未来的图景已经清晰可见。个人AI代理不再是科幻概念，而是即将到来的现实。

准备好迎接这个由AI代理驱动的新世界了吗？这场变革已经开始，而你我都将是其中的一部分。
